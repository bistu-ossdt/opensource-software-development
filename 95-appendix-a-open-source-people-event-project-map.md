# 附录 A 开源人物-事件-项目对照表

这一附录不是人物传记，也不是完整历史叙述，而是一张导航图。它把关键人物、代表性事件与核心项目或组织并列放在一起，帮助读者快速看见三条主线之间的关系：谁提出了关键问题，什么事件改变了开源历史的方向，哪些项目或组织把这些变化变成了可持续的实践。

与正文相比，这里更强调“对照”和“定位”，而不是完整展开；与后面的两份附录相比，这里更像是一张索引图，而不是展开论述。读者如果先记住的是人物，可以从这里反查其对应事件与项目；如果先记住的是某个历史事件，也可以回到这里看它由谁推动、落在什么对象上。

需要说明的是，这里选入的人物并不追求“所有著名程序员的总表”，而优先选择那些对开源的历史形成、制度表达、治理实践、协作基础设施和维护方法具有长期影响的人物。

## 5. 使用方式

这一附录最适合承担三种用途：

- 作为阅读正文时的快速导航页，帮助把人物、事件和项目重新连到一起。
- 作为进入附录 B 与附录 C 之前的总览页，先建立脉络，再进入展开阅读。
- 作为课程讨论时的索引页，帮助区分“谁提出问题”“谁推进制度”“谁提供基础设施”“谁总结方法”。

若想先建立时间脉络，可继续阅读“附录 B 开源历史关键事件时间线”；若想从某个人物出发更完整地理解其历史位置，可继续阅读“附录 C 开源人物谱”；若想查清正文中涉及的稳定概念，应回到“全书术语表”；若想追踪原始来源和官方材料，应回到“全书参考文献”。
