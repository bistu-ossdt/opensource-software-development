## 开源软件的发展

> 李宁，约3万字

在20世纪的50、60年代，计算机刚开始使用的时候，软件尚未成为独立的商品，人们购买的是计算机，软件只是作为硬件的附属品供人们来使用。人们用的最多的软件是操作系统和程序编译器。当时开发这些软件的主要是一些学术和研究机构，他们并不依靠开发软件来挣钱，当他们完成了某个软件的时候，往往会附上源代码。其中一个很重要的原因是当时的软件和硬件的依赖度很高，软件生产效率低下，往往存在很多Bug，所以需要经常修改，所以研究人员干脆就把一些通用的（称为公共域，Public Domain）软件的源代码公开，并鼓励大家帮助修改，移植到新的系统中去。例如1953年雷明顿兰德公司（Remington Rand） UNIVAC 部门就将他们开发的UNIVAC源代码提供给用户，并邀请他们将改进的代码再发给 UNIVAC。因此A-2被认为是第一个自由及开放源代码的软件。当时的很多计算机公司例如IBM和DEC开发的软件都带着源代码，还有一些大学的计算机实验室甚至规定，在他们的计算机上运行的软件必须都有源代码。
随着计算机系统越来越复杂，操作系统和编译器之类的软件开发成本也越来越高。计算机销售商通过软硬件捆绑将这部分成本分摊给用户，导致用户购买或租赁计算机设备的费用也越来越高。这时候用户反对软硬件捆绑的呼声日益高涨，妥协的结果是，一些软件仍可以免费使用，而另一些软件则要收费使用。这方面的一个例子是美国电话电报公司AT&T，他们70年代初向政府和学术机构免费分发UNIX操作系统，却在80年代初停止了UNIX的免费分发，如果要使用新的系统架构，则要购买商业许可证。IBM也不在为软件发行提供源代码。
之后，软件逐渐成为硬件一样真正的商品。几乎与此同时也出现了开源软件运动。下面以开源理论的形成与发展、开源操作系统与工具平台、开源云计算平台、开源人工智能平台、开源软件的商业化推动为主线，叙述开源软件的发展脉络及其特点。

### 开源运动的形成与发展

20世纪的80年代初，随着软件成为了商品，许多公司按许可证收取软件费用，并通过著作权和商标等限制软件的随意使用。然而，还有相当一部分人无视这些做法，我行我素，导致当时以黑客为代表的自由软件界和以Microsoft（微软）公司为代表的商业软件界的决裂。事情的导火索是1976年Microsoft比尔·盖茨的一篇文章《致业余爱好者的公开信》，他在文中对业余者大量使用Microsoft的Altair BASIC产品而不支付费用表示悲哀和愤怒，斥责他们是窃贼，呼吁把他们踢出电脑爱好者的阵营。
这时，以理查德·斯托曼（Richard Stallman）为代表的黑客群体挺身而出，他们认为这种商业化的做法，阻碍了软件的进步，剥夺了人们了解软件的权利，是违背道德的。当时的黑客（Hacker）是一群计算机编程高手，经常做一些惊世骇俗的事情，例如侵入要害系统以炫耀自己的技术，但是他们并不做坏事，而是善意地把发现的漏洞提供给相关人员以改进系统，这与今天专做坏事的“黑客”（Cracker）的概念是不同的。
斯托曼时任美国国家工程院院士，是当时的一位著名黑客。1984年，他试图构建一个自由、免费的操作系统生态环境，并取名为GNU（GNU’s Not UNIX），以示和当时收费的UNIX操作系统对立。斯托曼在1985年发布了GNU宣言，使用自由软件（Free Software）来定义他发起的GNU项目。1986年斯托曼创建了自由软件基金会（Free Software Foundation, FSF），主要目的是为开发GNU募集资金。1989年FSF发布了GNU通用公共许可证的第一版，也称为“Copy Left”，以保障GNU能够永久地免费共享、自由使用以及修改发布。
正是在这样的背景下，1996年蒂姆·奥莱利（Tim O'Reilly）（O'Reilly Media公司的创始人）召集了黑客社区中的重要成员：林纳斯·托瓦兹（Linus Torvalds）、三个主流脚本语言（Perl/Tcl/Python）的发明人、Netscape公司代表、Apache维护者等重量级人物一起开了个讨论会。重要成果之一是明确了不论是自由软件，还是开源软件，最重要的共性便是公开源码。这便是“开源”这个词的由来。
1997年，《新黑客字典》（The New Hacker Dictionary）的作者、程序员埃里克·雷蒙德（Eric S. Raymond）发表了名为“大教堂和集市”的文章，对比了商业开发模型以及开放源码开发模型，这是对开源的一个重要理论贡献。
1998年2月，埃里克·雷蒙德成立了开放源代码促进会，从事开放源代码的定义以及认证工作。

### 开源操作系统与平台软件的发展

本节将以开源软件中资源最丰富的UNIX/Linux操作系统与工具、面向Web的平台与工具，以及移动操作系统Android为主线介绍开源软件的发展历程。

### UNIX、Linux操作系统与工具

1980年代，斯托曼推动的GNU项目不断发展。GNU的目标是构造一个操作系统环境，但由于设计过于复杂，它的系统核心（GNU Hurd）的开发工作一再被推迟，而其他的组件在1991年都基本完成了，其中包括GNU编译器套装、GNU Debugger和GNU Emacs（由斯托曼本人开发），可谓“万事俱备，只欠东风”。恰巧当年芬兰赫尔辛基大学的学生托瓦兹基于GNU GPL框架发布了GNU/Linux，Linux便和其他GNU的组件结合，诞生出第一个真正属于自由软件的操作系统，至此，开源软件开始成长壮大，逐步走向正轨。
除了操作系统，80年代开源软件产品主要集中在一些基础工具软件上，例如：文本编辑器（Emacs）、编译器（gcc、lex/yacc）、构建工具（make）、版本控制工具（如DCVS）和缺陷跟踪工具（如Bugzilla）等等。那时起，开源编程语言变得十分丰富。直到今天，几乎所有的编程语言都有开源的实现。例如，1991年出现的Python，1995年出现的Ruby等等。2007年Sun Microsystems（太阳微系统公司）按照GNU许可证开放了Java语言开发工具包OpenJDK。另外，版本控制系统是开源开发不可缺少的工具，分布式版本控制系统（Distributed version control system，DVCS）从并发版本控制系统CVS（Concurrent Versions System）演变而来，2005年出现的Git是最为流行的DVCS，也是今天开源托管网站GitHub的基础。开源项目经常使用的缺陷追踪工具如Bugzilla和Redmine等也是这个阶段的产物。
Linux以往最受诟病的是用户界面过于专业，大多为命令行形式，令非专业人士望而却步。在90年代后期，基于X-Windows图形系统的桌面端应用环境快速发展，出现了著名的桌面环境如KDE和GNOME以及Qt和GTK工具包，此外还成立了致力于桌面端互操作标准化的组织freedesktop.org，大大提高了Linux桌面端使用的用户友好性，带动了Linux的应用普及。2011年4月发布的GNOME 3大幅改善了桌面的使用方式，优雅独特，效率更高，用户友好性更好，也为其他类似产品的设计带来了灵感。
值得一提的还有，2000年10月，Sun根据GNU较宽松通用公共许可证（GNU Lesser General Public License，LGPL）发行了StarOffice，其免费软件版本命名为OpenOffice.org，从此有了可与Microsoft Office媲美的开源办公套件。
从国内的发展状况看，1997年前后是开源软件的起步时间。我国最初关于开源软件的研究，主要是对Linux的汉化工作。随着开源的发展，出现了包括Linux虚拟服务器（LVS，Linux Virtual Server）、启动管理器（Smart Boot Manager, SBM）以及Mini GUI在内的诸多开源软件，这使得那个时期成为了我国开源软件发展史上的第一个高峰。

阅读：UNIX的历史
改编自——史蒂文·韦伯. 开源的成功之路[M]. 外语教学与研究出版社, 2007.
UNIX操作系统是美国AT&T公司1971年在PDP-11小型计算机上运行的操作系统。具有多用户、多任务的特点，支持多种处理器架构，最早由肯·汤普逊（Kenneth Lane Thompson）、丹尼斯·里奇（Dennis MacAlistair Ritchie）和道格拉斯·麦克罗伊（Douglas McIlroy）于1969年在美国电报电话公司（AT&T）的贝尔实验室开发。
UNIX的诞生和Multics（Multiplexed Information and Computing System）有一定的渊源。Multics是由麻省理工学院、AT&T贝尔实验室和通用电气合作进行的操作系统项目，被设计运行在GE-645大型主机上，但是由于整个目标过于庞大，糅合了太多的特性，Multics虽然发布了一些产品，但是性能都很低，最终以失败而告终。
AT&T最终撤出了投入Multics项目的资源，其中一位开发者汤普逊则继续为GE-645开发软件，并最终编写了一个太空旅行游戏。经过实际运行后，他发现游戏速度很慢而且耗费昂贵——每次运行要花费75美元。
在里奇的帮助下，汤普逊用PDP-7的汇编语言重写了这个游戏，并使其在DEC PDP-7上运行起来。这次经历加上Multics项目的经验，促使汤普逊开始了一个DEC PDP-7上的新操作系统项目。汤普逊和里奇领导个开发者组，开发一个新的多任务操作系统。这个系统包括命令解释器和一些实用程序。在1970年，PDP-7只能支持两个用户，有人就开玩笑地戏称他们的系统其实是“UNiplexed Information and Computing System”，缩写为“UNICS”。于是这个项目被称为UnICS（Uniplexed Information and Computing System）。后来大家取其谐音改成了UNIX这个名字。
最初的UNIX是用汇编语言编写的，一些应用是由称为B语言的解释型语言和汇编语言混合编写的。B语言在进行系统编程时不够强大，1973年汤普逊和里奇用C语言重写了UNIX。在当时，为了提高效率，系统程序都用汇编语言编写，所以汤普逊和里奇此举是极其大胆而有创新意义的。用C语言编写的UNIX代码简洁紧凑、易移植、易阅读、易修改，为此后UNIX的发展奠定了坚实基础。
1974年，汤普逊和里奇合作在ACM通信上发表了一篇关于UNIX的文章，这是UNIX第一次出现在贝尔实验室以外。此后UNIX被政府机关，研究机构，企业和大学注意到，并逐渐流行开来。
1975年，UNIX发布了4、5、6三个版本。1978年，已经有大约600台计算机在运行UNIX。1979年，UNIX版本7发布，这是最后一个广泛发布的、供研究的UNIX版本。20世纪80年代相继发布的8、9、10版本只授权给了少数大学。此后这个方向上的研究导致了Plan 9的出现，这是一个新的分布式操作系统。
在UNIX产生后的10年中，UNIX在学术机构和大型企业中得到了广泛的应用。当时的UNIX拥有者AT&T公司以低廉甚至免费的许可将UNIX源码授权给学术机构用于研究或教学，许多机构在此源码基础上加以扩充和改进，形成了UNIX变种，这些变种反过来也促进了UNIX的发展。其中最有影响的是加州大学伯克利分校开发的BSD版本的UNIX，被很多商业厂家采用，成为很多商用UNIX的基础。后来AT&T意识到了UNIX的商业价值，1979年版本7发布之后不再将UNIX源码授权给学术机构，并对之前的UNIX及其变种声明了版权。
1982年，AT&T基于版本7开发了UNIX System III的第一个版本，这是一个商业版本，仅供出售。为了解决混乱的UNIX版本情况，AT&T还综合了其他大学和公司开发的各种UNIX开发了UNIX System V Release 1。这个版本不再包含源代码。
由于AT&T的UNIX System V不再免费提供源代码，加州大学柏克莱分校继续开发BSD UNIX，作为UNIX System III和V的替代选择。BSD对UNIX最重要的贡献之一是TCP/IP。BSD有8个主要的发行版中包含了TCP/IP：4.1c、4.2、4.3、4.3-Tahoe、4.3-Reno、Net2、4.4以及 4.4-lite。这些发布版中的TCP/IP代码几乎是现在所有系统中TCP/IP的先驱，包括AT&T System V UNIX 和Microsoft Windows。
BSD UNIX不断增大的影响力终于引起了AT&T的关注，于是带来了一场旷日持久的版权官司，这场官司一直打到AT&T将自己的UNIX系统实验室卖掉。新接手的Novell公司采取了一种比较开明的做法，允许伯克利分校自由发布自己的BSD版本的UNIX，但是前提是必须将来自于AT&T的代码完全删除，于是诞生了4.4 BSD Lite版。由于这个版本不存在法律问题，4.4 BSD Lite成为了现代BSD系统的基础版本。尽管后来非商业版的UNIX系统又经过了很多演变，但其最终都是建立在BSD版本上（Linux除外）。所以从这个角度看，4.4 BSD是所有自由版本UNIX的基础，它们和System V及Linux等共同构成UNIX操作系统这个大家族。
BSD在发展中也逐渐派生出3个主要的分支：FreeBSD、OpenBSD和NetBSD。
在当时，一些公司为其自己的小型机或工作站提供商业版本的UNIX系统，有些选择System V作为基础版本，有些则选择了BSD。BSD的一名主要开发者，比尔·乔伊（Bill Joy），在BSD基础上开发了SunOS，并最终创办了Sun Microsystems。
同时AT&T继续为UNIX System V增加了文件锁定、系统管理、作业控制、流和远程文件系统。1987到1989年，AT&T决定将XENIX（Microsoft开发的一个X86-PC上的UNIX版本），BSD，SunOS和System V融合为System V Release 4（SVR4）。这个新发布版将多种UNIX版本的特性融为一体，结束了混乱的竞争局面。
UNIX System V Release 4发布后不久，AT&T就将其UNIX出售给了Novell。Novell期望以此来对抗Microsoft的Windows NT，但Novell当时的市场出现了问题，最终Novell将SVR4的权利出售给了X/OPEN Consortium，后者是定义UNIX标准的产业团体。最后创建了Open Group，将X/OPEN和OSF/1（两种UNIX标准）合并。另一方面，实际的UNIX代码则辗转到了Santa Cruz Operation（SCO）公司，这家公司后来出售给了Caldera Systems。Caldera原来也出售Linux系统，交易完成后，新公司又被重命名为SCO Group。
此后的几十年中，UNIX仍在不断变化。很多大公司在取得了UNIX的授权之后，开发了自己的UNIX产品，比如IBM的AIX、HP的HP-UX、SUN的Solaris和SGI的IRIX。UNIX因为其安全可靠，高效强大的特点在服务器领域得到了广泛的应用。在GNU/Linux流行开始前，UNIX也是科学计算、大型机、超级计算机等的操作系统的主流。
UNIX不仅仅是一个操作系统，更是一种生活方式。经过几十年的发展，UNIX在技术上日臻成熟的过程中，其独特的设计风格也深深地吸引了一大批技术人员，他们在维护、开发、使用UNIX的同时，UNIX也影响了他们的思考方式和看待世界的角度。

阅读：Linux的历史
改编自——史蒂文·韦伯. 开源的成功之路[M]. 外语教学与研究出版社, 2007.
Linux内核最初只是由芬兰人林纳斯·托瓦兹（Linus Torvalds）在赫尔辛基大学上学时出于个人爱好而编写的。当时他不满意Minix这个教学用的操作系统，因为它只能在有限硬件上运行。托瓦兹最初的设想中，Linux是一种类似Minix的操作系统。Linux的第一个版本在1991年9月被赫尔辛基大学FTP服务器的管理员阿里·莱姆克（Ari Lemmke）发布在Internet上，托瓦兹最初称它为“Freax”，含有自由（“free”）和奇异（“freak”）两种含义，并且附上了“X”这个常用的字母，以表明它是一个“UNIX-like”系统。但是莱姆克嫌“Freax”不好听，照着托瓦兹的名字改成了“Linux”。当时Linux仅有10000行代码，需运行于Minix操作系统之上，并且必须使用硬盘开机。随后托瓦兹在10月份发布了第二个版本（0.02版）。同时这位芬兰赫尔辛基的大学生在UNIX User Group “comp.os.minix”上发布一则消息：
“Hello everybody out there using minix- I'm doing a (free) operation system (just a hobby, won't be big and professional like gnu) for 386(486) AT clones.”（各位使用Minix的人，大家好，我正在为386（486）系列电脑开发一个（免费的）操作系统（纯属兴趣爱好，不会做得像GNU那么高大上）。

### 后来Linux的命运和GNU紧密联系到了一起。

1983年，斯托曼创立了GNU计划。这个计划的目标是形成一个完全自由的类UNIX操作系统环境。自1990年代发起这个计划以来，GNU大量开发和收集了该系统所需要的各种组件，包括：函数库、编译器、调试工具、文本编辑器、网页服务器以及用户界面，只是当时内核仍然付阙。这是因为GNU原计划基于马赫微核（Mach microkernel）架构开发系统内核，即所谓的GNU Hurd，但是这个设计异常复杂，开发进度异常缓慢。
直到1991年Linux 内核发布的时候，GNU已经几乎完成了除了系统内核之外的各种必备组件的开发。在托瓦兹和其他开发人员的努力下，这些GNU组件得以在Linux内核上运行。整个系统基于GNU通用公共许可证，即GPL（GNU General Public License），但是Linux内核当时并不是GNU计划的一部分。1994年3月Linux 1.0版正式发布，代码量为17万行，当时按照完全自由免费的协议发布，随后正式采用GPL协议。至此，Linux的代码开发进入良性循环。很多系统管理员开始尝试使用Linux，并将修改的代码提交给核心小组，因而Linux的代码中加入了对不同硬件的支持，大大提高了跨平台的可移植性。
Linux的标志和吉祥物是一只名字叫Tux的企鹅，听说其灵感来源于托瓦兹在澳大利亚时被一只动物园里的企鹅咬了一口的经历，不过更好的解释是：企鹅代表南极，而南极又是全世界所共同拥有的一块陆地，意味着Linux是所有人的Linux。Linux的注册商标归托瓦兹所有，这个原因是，1996年一个名字叫做威廉·克罗齐（William R. Della Croce）的律师开始给各个Linux发行商写信，声明他拥有Linux商标的所有权，并且要求各个发行商支付版税。结果这些发行商集体上诉，要求将该注册商标重新分配给托瓦兹。托瓦兹一再声明Linux永远是大家的，他本人可以被卖掉，但Linux绝不能卖。
Linux诞生于Internet，发展于Internet，自然具有强大的网络功能。Linux可以很好地支持TCP/IP协议，能够与Windows、UNIX、Novell、MacOS等不同操作系统集成在一起，相互共享资源，还可以通过Modem、ADSL、ISDN或各种专线直接连接到Internet上。Linux不仅能够作为网络工作站使用，作为各类网络服务器更是得心应手，功能强大而且性能稳定。常被用于文件服务器、打印服务器、数据库服务器、Web服务器、邮件服务器、FTP服务器、新闻服务器、代理服务器、认证服务器，以及路由器、集群、网关等各种网络设备中。
Linux不仅具有类似DOS的命令行操作界面，还具有功能强大的图形界面。它与DOS有着本质的不同。DOS只是一个单用户单任务的操作系统，即一台电脑上的操作系统同一时刻只允许一个用户运行一个程序，而Linux系统则允许多个用户在本地或远程同时登录到系统上，分别运行不同的程序，当然也允许一个用户同时运行多个不同的程序，且可以方便地在不同用户或不同程序之间切换。而Microsoft公司的操作系统到了Windows NT版本以后才真正实现了多用户多任务。
GNU/Linux发展迅速。从普通的PC机到高端的超级并行计算机系统，都可以运行Linux系统。Linux符合IEEE POSIX标准，特别注重可移植性，使UNIX下的许多应用程序可以很容易地移植到Linux平台上，相反也是这样。
Linux本来是一个操作系统的核心而不是一个完整的操作系统，因为它缺少许多必要的外围工具软件。Linux盈利的方式一般是将Linux核心、GNU软件包、工具集和文档、桌面环境等打包在一起，形成一个完整操作系统套件，称为Linux发行版。Linux发行版一般易用性较好，可以大幅降低用户的使用和维护成本，为Linux的普及发挥了重要作用。1994年马克·尤因（Marc Ewing）成立了Red Hat软件公司，成为最著名的Linux发行版分销商之一。中国的第一个Linux发行版红旗Linux于1999年8月发行。
1997年夏大片《泰坦尼克号》在制作特效中使用的160台Alpha图形工作站中，有105台采用了Linux操作系统。1998年4月Mozilla 代码发布，成为Linux图形界面上的主流浏览器。1999年，发布了Linux 2.2以及GNOME 1.0；同年支持Linux 2.2的Red Hat 6.0发布。包括Sun、HP、IBM等在内的诸多知名公司开始采用或销售安装了Linux系统的计算机，几个主要的软件厂商，包括CA，Corel，Sun，HP，IBM，Informix，Interbase，Oracle以及Sybase都宣布了支持Linux的产品计划。IBM宣布与Redhat公司建立伙伴关系，以确保Redhat能在IBM机器上运行。这些大公司的加入使开源运动更加活跃。2000年Red Hat发布了嵌入式Linux开发环境，Linux在嵌入式行业的潜力逐渐被发掘出来。2001年，Oracle 发布了Oracle 9i的Linux版本；IBM投入10亿美元扩大Linux系统的运用。2002年3月，新的Linux版本支持64位的计算机。2003年1月，NEC宣布在其手机中使用Linux操作系统，代表着Linux成功进军手机领域。2018年10月，IBM以340亿美元的价格收购RedHat，IBM表示将把RedHat作为一个独立的业务部门来运作，继续原来的运行方式，而不受母公司文化的影响。

### 面向Web的平台与工具

Linux伴随着Internet诞生和成长，但那时的Internet并不普及。80年代后期，Internet逐渐兴盛起来。一些互联网应用例如BBS、Usenet和UUCP Net加速了开源软件的发展。这些应用加强了开源社区的联络，为程序员提供了便捷的方式来共享代码或帮助他人编写软件。
1980年代美国加州大学伯克利分校开发的BIND（Berkeley Internet Name Domain）一直是互联网上用得最多的开源域名解析服务器程序。
1994年，Netscape 公司成立。凭借其独创的JavaScript编程语言和浏览器渲染引擎，一度占据了90%以上的浏览器市场份额。然而，Microsoft也瞄准了网络浏览器这个市场。通过在 Windows中内置Internet Explorer（IE）浏览器，Microsoft快速蚕食着Netscape的市场份额。特别是1998年，Microsoft发布的 Windows 98中内置的 Internet Explorer 4.0从功能上压倒了Netscape的产品。岌岌可危的 Netscape在1998年做了一个大胆的决定：公开Netscape网络浏览器的源码（日后它重生为Firefox，与IE展开新的争夺）。
1990年代开源软件出现了一个重量级的应用——Web服务器Apache。Apache从当时最流行的HTTP服务器NCSA HTTPd 1.3发展而来，它稳定、高效、灵活、具有扩展性，具有很高的性价比。很快大约60%的网站都使用了Apache，成了全球互联网服务商的首选平台。直到2015年，结合Linux内核，Apache HTTP与MySQL数据库引擎以及PHP网页编程语言一起被称为LAMP系统（意指互联网开发的“明灯”）。实际上，在1990年代中后期，Perl语言早于PHP并主导着网络编程，因此。LAMP也可以认为是Linux，Apache HTTP、MySQL和Perl的组合。时至今日，Linux和MySQL已经是地球上大多数数据中心的标配。而Apache更是成就了巨无霸式的开源基金会，拥有1.9亿行代码。
2000年出现了互联网经济泡沫。2000年3月10日，当日纳斯达克股指到达5000多点的峰值，随后大幅下跌。许多公司被严重地高估市值，在股价崩盘时，不得不走向破产。大多数互联网公司也在把风投资金烧光之后倒闭，许多甚至从来没有盈利过。据统计，互联网经济泡沫在两年间抹去了互联网技术公司约5万亿美元的市值。
经济泡沫给Linux和开源软件也带来了影响，使得Red Hat这样的公司的股价也大幅下跌。整个世界对于开源似乎都失去了兴趣。但是这种状况并没有维持太久，从2005年开始，新一轮的开源软件又继续发展起来。
2005年之后，以Google为代表的大型互联网企业，开始以各种方式发布开源软件，最为著名的是Chrome浏览器以及Android操作系统。此外还出现了几种有重要影响的开源软件，如浏览器引擎WebKit。WebKit原是由Apple（苹果公司）发起的开源项目，从早期KDE系统的KHTML和KJS两个子系统基础上发展而来。后来成为苹果Safari浏览器和GoogleChrome浏览器使用的核心引擎。其他的例如：Sendmail（电子邮件服务器）、Apache Tomcat（Web服务器）、PHP（脚本语言）、MySQL（关系型数据库）、Eclipse（集成开发环境）、FireFox（浏览器）、Lucene（检索引擎）、Hadoop（面向大数据的分布式架构）、Unity（实时内容开发平台）、R（统计编程语言）、Node.js（JavaScript运行时环境）。
2013年5月，Facebook（脸书）公司开源了React项目，React是一个用于构建用户界面的JavaScript 程序库。由于其设计思想独特，富于创新，性能出众，代码逻辑简单，得到越来越多的人的关注，被认为可能是将来Web开发的主流工具。
2014年10月，Google（谷歌）公司发布Angular。它是一个基于TypeScript的开源的Web应用设计框架与开发平台，用于创建高效、复杂、精致的单页面应用。由Google的Angular团队以及社区共同领导。
Microsoft在2015年4月发布了Visual Studio Code开源项目。它是一个运行于 Mac OS X、Windows和Linux之上的，针对于编写现代Web和云应用的跨平台源代码编辑器。
国内在2005年之后也出现了为数不多的一些开源软件项目，如清华大学陈渝主持的SkyEye，该项目旨在提供一个面向嵌入式软件开发的仿真器（虚拟机）。持续活跃长达七年之久，吸引了许多来自海外的高手参与。此后，淘宝、新浪、百度、腾讯和华为等也陆续加入了开源的行列，出现了一些优秀的产品，例如分布式对象存储系统和海量数据管理系统TFS，TAIR和OceanBase等。随着“开源中国”等社区的兴起，开源软件逐渐多了起来。据统计，当前已经有一千多个由国人开发或者主持的开源软件。这和十年前相比，已经有了非常大的进步。

### 移动操作系统Android

Android（安卓）是一种基于Linux内核（不包含GNU组件）的开源操作系统。主要用于移动设备，如智能手机和平板电脑，由Google和开放手机联盟领导及开发。Android操作系统最初由安迪·鲁宾（Andy Rubin）开发，主要支持手机。2005年8月由Google收购。2007年11月，Google正式向外界展示Android操作系统。Android的上层是虚拟机和Java应用，底层包括大量开源软件，如Linux操作系统内核、SQLite内嵌式数据库、FreeType矢量字体渲染库等等。同时，Google与84家硬件制造商、软件开发商及电信运营商组建开放手机联盟共同研发和改良Android系统。随后Google基于Apache许可证发布了Android的源代码。第一部Android智能手机HTC G1于2008年9月发布，同月Google发布Android 1.0。Android逐渐扩展到平板电脑及其他领域上，如电视、数码相机、游戏机和智能手表等。2011年第一季度，Android在全球的市场份额首次超过塞班系统，跃居全球第一。2013年第四季度，Android平台手机的全球市场份额已经达到78.1%。2013年09月24日在Android发布5周年之际，全世界使用这款系统的设备数量达到了10亿台。
Android在正式发行之前，最开始拥有两个内部测试版本，并且以著名的机器人名称来对其进行命名，它们分别是：阿童木（Android Beta），发条机器人（Android 1.0）。后来由于涉及到版权问题，Google改为用甜点的名字命名它们的系统版本。甜点命名法始于Android 1.5版。作为每个版本代表的甜点的尺寸越变越大，然后按照26个字母数序：纸杯蛋糕（Android 1.5），甜甜圈（Android 1.6），松饼（Android 2.0/2.1），冻酸奶（Android 2.2），姜饼（Android 2.3），蜂巢（Android 3.0），冰激凌三明治（Android 4.0），果冻豆（Jelly Bean，Android4.1和Android 4.2），奇巧（KitKat，Android 4.4），棒棒糖（Lollipop，Android 5.0），棉花糖（Marshmallow，Android 6.0），牛轧糖（Nougat，Android 7.0），奥利奥（Oreo，Android 8.0），派（Pie，Android 9.0）。从Android 10开始，Android不再按照食品的字母顺序命名，而是改回版本号，就像Windows和iOS系统一样。
除了操作系统之外，Android还是一个开放的生态环境，它包括操作系统、用户界面和应用程序，这包括了移动电话工作所需的全部软件，并迅速发展成为智能移动终端的操作系统。配合Android，Google推出的Android Market平台可以让应用程序开发者轻松地进入这个市场，手机用户可在该平台寻找、购买和下载Android操作系统上的应用程序，也可在Android Market上销售自己的软件，并通过交易获得利润。Android独到的开放性和可移植性获得了广大社区的支持，打败了曾经流行的塞班、黑莓等对手，逐步成为最流行的操作系统，今天Android已经拿下了智能手机操作系统的70%市场份额。
如果说2000年代是开源与闭源世界竞争的年代，那么2010年代就是开源胜出并开始推动技术创新的10年。最近10年，开源软件的发展出现几个新的特征：首先很多大公司积极推动着开源软件的发展，产生了一些重量级的产品，如Microsoft Visual Studio Code、Facebook React、Google TensorFlow等等；其次开源软件与云计算、大数据和人工智能紧密结合，呈现出智能时代的信息技术特征。

### 开源云计算平台的发展

2006年前后，Apache基金会发布了Hadoop。它是一个分布式系统基础架构。用户可以在不了解系统底层细节的情况下，开发分布式程序。充分利用集群优势进行高速运算和存储。Hadoop框架最核心的设计是HDFS和MapReduce。HDFS（Hadoop Distributed File System）是一个分布式文件系统，有高容错性的特点，可部署在廉价的硬件上，提供高吞吐量来访问应用程序的数据，适合那些有着超大数据集的应用程序。MapReduce是一种编程模型，用于大规模数据集的并行运算。借助Map和Reduce的机制，极大地方便了编程人员在不熟悉分布式并行编程的情况下，将自己的程序运行在分布式系统上。
2010年开源的OpenStack是一个云计算管理平台项目，是一系列开源软件的组合。由美国国家航空航天局（NASA）和Rackspace合作研发并发起，以Apache许可证授权的开源项目。超过半数的企业选择OpenStack，并采用商业化的技术服务支持。
2013年10月dotCloud基于Apache 2.0发布Docker，它是一个使用Go语言的应用容器引擎，源代码托管在Github上。Docker应用非常广泛，它使用客户端-服务器（C/S）架构模式，使用远程API来管理和创建Docker容器。Docker容器通过Docker镜像来创建。它可让开发者把应用程序打包到一个可移植的镜像中，然后发布到任何Linux或Windows机器上，也可以实现虚拟化。
2014年Google基于Apache 2.0许可证发布Kubernetes，它是一个开源工具，用于管理云平台中多个主机上的容器化应用，Kubernetes的目标是简单、高效地部署容器化的应用。Kubernetes提供了部署、规划、更新、维护应用的机制。Kubernetes对企业级业务的发展产生了深远的影响。
大约在2016年左右，云计算在IT行业得到全面应用。据2018年统计，超过七成的企业已经使用容器技术或正在测试应用环境。近六成的企业选择Docker作为容器运行的引擎以支持快速弹性扩容和高移植性。
其他的开源云计算设施包括：Spring Cloud是一系列框架的有序集合。它利用Spring Boot的便利性简化了分布式系统基础设施的开发工作，如服务发现注册、配置中心、消息总线、负载均衡、断路器、数据监控等，都可以用Spring Boot的开发风格做到一键启动和部署。超过六成的企业选择Spring Cloud 作为其微服务框架。Jenkins是一个基于Java开发的开源持续集成工具，用于监控持续重复的工作，旨在提供一个开放易用的软件平台，使软件的持续集成变为可能。它是目前企业使用最广泛的开源集成工具。

### 开源人工智能平台的发展

开源软件很早就在多媒体处理、机器视觉、模式识别和机器学习等方面计类了丰富的资源。例如，2000年Intel发布了跨平台计算机视觉库OpenCV，实现了图像处理和计算机视觉方面的很多通用算法，极大提高了程序的可移植性和开发效率，在人机五动、人脸别识别、图像拼接和运动跟踪等人工智能领域得到了广泛应用。
近二十年来人工智能重新又重返历史舞台。2016年DeepMind研发的AlphaGo 以4:1 战绩战胜世界围棋冠军李世乭，再将人工智能推向了新的高度。人工智能开源软件从诞生至今经历了多个发展阶段，已在自然语言处理、计算机视觉、智能语音、无人系统、知识图谱、信息安全、虚拟现实等领域得到深度的应用。近年来，人工智能开源软件蓬勃发展，已成为发达国家政府、国内外顶尖学术研究机构、跨国技术全球企业、国际开源机构等组织的研究的热点，备受产业界广泛关注和积极投入。
本节将重点介绍主要的开源机器学习框架和自然语言处理工具。

### 开源机器学习框架

随着深度学习领域不断取得突破，深度学习算法如CNN、RNN、LSTM等在诸多领域的表现优异，越来越多的机构和个人加入到深度学习的研究领域。深度学习在图像分析、语音识别、自然语言处理、视频分类等诸多领域性有极高的研究价值和商业价值，许多世界顶尖高科技公司相继推出了自己的开源深度学习框架，如Google的TensorFlow、Bert，Apache的MXNet，Facebook的Pytorch和Microsoft的CNTK等，甚至英伟达这样的硬件厂商都推出了RAPIDS这样的开源人工智能框架。形成了异彩纷呈的局面。

### 图 2-X 主流开源机器学习框架的发展脉络

2007年，数据科学家大卫·库尔纳波（David Cournapeau）发起了开源机器学习项目Scikit-learn，并在2010年基于BSD许可证发布了第一版。Scikit-learn是基于Python语言的简单高效的数据挖掘和数据分析工具，包含了几乎所有常见的机器学习算法和模型。
2009年，Apache基于Apache 2.0许可证发布了基于Java语言的开源机器学习框架Mahout。它的最大特点是使用Apache Hadoop库，可以有效地扩展到Hadoop集群，解决并行挖掘的问题。Mahout实现了许多可扩展的经典机器学习算法和模型，尤其适合用于聚类、分类、推荐过滤等领域。
2010年，Apache基于Apache 2.0许可证发布了构建在Apache Spark之上的开源机器学习项目MLlib。它的最大特点是可以与Spark SQL、GraphX、Spark Streaming无缝集成，非常适合分布式大数据处理。MLlib实现了机器学习常见算法，包括分类、聚类和逻辑回归等，它将Spark的分布式计算成功应用到机器学习领域。
2014年，美国加州大学伯克利分校中国籍博士贾扬清基于BSD许可证发布了开源软件Caffe，它是一种深度学习框架，具有上手快、速度快、模块化和开放性好等特点，支持GPU计算，可以在CPU和GPU之间直接无缝切换，被广泛应用于计算机视觉、自然语言处理、语音识别等多个领域。
2015年，Google基于Apache 2.0许可证发布深度学习开源框架Tensor Flow，它是在DistBelief（Google的另一个机器学习程序库）的基础上研发的第二代人工智能学习系统。Tensor Flow具有高度的灵活性和可移植性等特点，能很好地支持各种深度学习算法，包括卷积神经网络、循环神经网络和长短期记忆网络等深度神经网络模型，提供了丰富的构建和训练机器学习模型的应用编程接口，支持经典机器学习算法；支持异构计算和分布式计算。Tensor Flow已被广泛地应用于语音识别或图像识别等多项机器学习和深度学习领域。此外，它还支持异构设备的分布式计算，能够在各个平台上运行，从手机、单CPU/GPU到上千个GPU组成的分布式系统。Tensor Flow是目前最受欢迎的深度学习框架之一。
2015年，Google工程师弗朗索瓦·乔利特（François Chollet）基于MIT许可证发布了Keras。它是一个用Python编写的开源人工神经网络库，可以作为TensorFlow、Microsoft-CNTK和Theano的高阶应用程序接口，进行深度学习模型的设计、调试、评估、应用和可视化。Keras支持现代人工智能领域的主流算法，包括前馈结构和递归结构的神经网络。在硬件和开发环境方面，Keras支持多操作系统下的多GPU并行计算，可以根据后台设置转化为TensorFlow、Microsoft-CNTK等系统下的组件。
2016年，北京大学的董豪在英国帝国理工大学读博期间开发了TensorLayer软件，并基于Apache 2.0许可证发布成开源软件。TensorLayer专为研究人员和工程师设计，建立在Google Tensor Flow之上，能与Tensor Flow全透明衔接，它提供高级别的深度学习应用程序编程接口，可以加快研究人员和工程师的开发速度，因而广受欢迎，荣获2017年ACM Multimedia年度最佳开源软件奖。TensorLayer具有易于使用、易于修改和易于扩展的优点，支持GPU计算，计算速度快，主要应用于多模式研究、图像转换和医疗信号处理等领域。
2016年，Microsoft基于MIT许可证发布了CNTK开源软件。它是一个人工智能工具包，具有速度快、可扩展、稳定可靠以及与C++和Python兼容等优点，支持各种神经网络模型、异构计算、分布式计算，且性能优异，在语音识别、机器翻译、类别分析、图像识别、图像字幕、文本处理、语言理解和语言建模等方面都有不少应用。Microsoft还使用它来支持Skype、Cortana和Bing中的智能处理。
2017年Facebook基于BSD许可证发布了PyTorch。它是在Python语言环境中使用的机器学习框架，PyTorch的前身是Torch，它不仅能够实现强大的GPU加速，同时还可支持动态神经网络，这是很多主流深度学习框架比如TensorFlow等做不到的。
2016年，百度基于Apache 2.0许可证发布其开源深度学习平台Paddle Paddle。Paddle Paddle起源于百度于2013年自主研发的深度学习平台，是一个相对全功能的深度学习框架。Paddle Paddle已实现CPU/GPU单机和分布式模式，同时支持海量数据训练以及多台机器并行运算，可以轻松应对大规模的数据训练。此外，Paddie Paddle具有易用、高效、灵活和可伸缩等特点。Paddle Paddle使百度成为继Facebook、Google和IBM之后世界上又一个提供开源深度学习框架的科技公司。国内除了百度的Paddle Paddle，腾讯的Angle等也应用广泛，具有较大的影响。
2020年2月Apache基于Apache2.0许可证发布了MXNet。它是一个开源的深度学习框架，用于训练和部署深度神经网络。它具有可扩展性，可以进行快速的模型训练，并支持灵活的编程模型和多种编程语言（包括C ++，Python，Java，Julia，Matlab，JavaScript，Go，R，Scala，Perl和Wolfram语言），MXNet库是可移植的，可以扩展到多个GPU和多台机器。

### 自然语言处理开源工具

自然语言处理（Natural Language Processing，NLP）是人工智能的基础。针对中外文的自然语言处理，涌现出大量的优秀开源软件。
2001年，美国宾夕法尼亚大学基于Apache协议发布了NLTK（Natural Language Toolkit），它提供了包括WordNet在内的百余个语料资源，以及分类、分词、词干提取、词性标注、依存分析、语义推断等一系列功能，还维护着一个活跃的开发者论坛。在众多大学课程中作为教学工具广泛使用。
2007年，加拿大蒙特利尔大学基于BSD许可证发布了Theano，它是是一个擅长处理多维数组的Python库，用于定义、优化、求值数学表达式，十分适合与其它深度学习库结合起来进行数据挖掘，高效地解决多维数组的计算问题。它设计的初衷是为了执行深度学习中大规模神经网络算法需要的运算。
2008年，拉迪姆·雷胡雷克（Radim Rehurek）基于GPL许可证发布了Gensim。它是一个基于机器学习的自然语言处理工具包，提供主题建模、文档索引以及相似度计算等功能。Gensim按流式处理数据，所实现的算法不受语料大小影响，可以处理大量的输入数据。
2010年，斯坦福大学自然语言处理组基于GPL协议发布了StanfordCoreNLP。它支持多种语言的处理，提供词干提取、词性标注、命名实体识别、依存语法分析、指代消解、情感分析、关系抽取等功能，还集成了很多自然语言处理工具，为多种主流编程语言提供开发接口，可以作为Web服务运行。
2010年，Apache基于Apache 2.0协议发布了OpenNLP，它是基于机器学习的自然语言处理工具包，提供基于机器学习的自然语言文本处理功能，包括分词、分句、分块、词性标注、命名实体识别、语法解析、指代消解等。它可为多种语言提供预训练模型以及对应的标注语料资源，既可以通过Java API或命令行接口调用，也可以作为组件集成到Apache Flink、NiFi、Spark等分布式流数据处理框架中。
2011年，我国哈尔滨工业大学社会计算与信息检索研究中心基于GPL许可证发布了LTP （Language Technology Platform）。它是一个支持Java和Python语言接口的中文处理基础平台，可以高效、准确地提供中文分词、调性标注、命名实体识别、依存句法分析、语义角色标注等自然语言处理功能。
2013年，Google基于Apache 2.0发布了开源工具Word2Vec，该工具利用深度学习的思想，通过训练将每个词映射成K维实数向量，把对文本内容的处理简化为K维向量空间中的向量运算。Word2Vec可以用来处理自然语言处理相关的诸多工作，比如聚类、同义词发现、词性分析等。
2014年，Explosion AI基于MIT许可证发布了spaCy。它是支持20多种语言的自然语言处理包，提供词性标注、依存分析、命名实体识别等功能，以支持企业级大规模应用和快速高效而著称。spaCy可用于自然语言深度学习的预处理阶段，与Tensorflow、PyTorch、Scikit-learn、Gensim等结合使用。
2014年，Global Vectors发布GloVe。它是一个分布式单词表示模型，也是一个获取单词向量表示的无监督学习算法。是通过将单词映射到有意义的空间来实现的，它在文本语料库上进行训练，并捕获全局单词-单词共现信息以进行单词嵌入，其中单词之间的距离与语义相似性有关。
2014年，复旦大学自然语言处理实验室基于LGPL 3.0许可证发布了FudanNLP。它是用Java语言编写的中文自然语言处理工具包，提供中文分词、词性标注、实体名识别、关键词抽取、依存句法分析、时间短语识别、文本分类、新闻聚类等功能，还提供了在线学习、层次分类和聚类等结构化学习算法。
2015年，中科院计算所发布了NLPIR汉语分词系统（又名ICTCLAS），NLPIR提供中文分词、词性标注、命名实体识别、微博分词、新词发现、关键词提取等功能。支持多种字符编码。
2016年，清华大学自然语言处理与社会人文计算实验室基于MIT许可证发布了THULAC中文词法分析工具包，包括C++、Java 、Python三个版本，主要提供中文分词和词性标注等功能，具有分词能力强、准确率较高、速度较快等特点。
2018年Google的雅各布·德夫林（Jacob Devlin）及其同事于发布了BERT（Bidirectional Encoder Representation from Transformers），它支持transformer的双向编码器表征。与之前的语言表征模型不同，BERT设计成通过在所有层中的左右上下文共同调节来对未标记文本的深度双向表征进行预训练。BERT可以用于问答系统，情感分析，垃圾邮件过滤，命名实体识别，文档聚类等任务中，作为这些任务的基础语言模型。BERT被认为给NLP带来了里程碑式的改变，也是NLP领域近期最重要的进展。
除上述的工具之外，还有很多相关的优秀开源软件，例如：用于知识图谱表示的开放资源OpenKE和Scikit-Kge，以及用于关系抽取的开放资源OpenNRE等，均有广泛应用。

### 开源软件的商业化推动

在传统的观点看来，开源软件和商业软件二者水火不容。但随着软件产业的发展，软件即服务的概念深入人心，开源软件和商业活动日渐结合起来，形成优势互补，达到双赢的目的。结合开源理念的一批商业项目如雨后春笋，生机勃勃；同时越来越多的传统商业软件公司逐渐认识到开源软件的价值，大规模投入到开源活动之中。开源的商业模式是整个商业社会走向开放和创新的一个桥梁，对于有效集中群体智慧，减低开发成本和建立黏性用户群体等方面，它都具有独特的优势。在2002年的一次对开源软件调查中，人们发现约有三分之一的开发者因开源开发领取报酬，他们对开源的贡献源自一些企业对开源软件提供的资助。无论出发点如何，这些企业对开源运动的贡献是巨大的，而开源软件也常常给企业带来了丰厚的回报。
众多的开源企业中RedHat是其中一个成功的例子，它免费提供产品的源码，通过技术支持和专业服务来盈利，2004年收入达到1.25亿美元。许多其他公司如JBoss、Mysql、Mozzilla（Mozilla基金会成立的企业）也都取得不俗业绩。
传统企业也非常积极地参与各项开源活动。IBM一度成为开源活动幕后最大的资助者，对许多开源项目都提供了资助。早在2000年IBM就声称已经为Linux捐赠超过10亿美元。同时它还不断给予开源社区技术资助，捐赠项目或源码给开源社区，开放了许多公司专利。2005年以来IBM向开源社区开放了500个软件专利，还在Source Forge上发布了30个开源项目。它还频繁收购小型开源企业，积极举办各种开源推广活动。Sun公司（2009年被Oracle公司收购）一贯支持开源软件，在几乎所有的产品线上均有开源项目，并发布了重量级开源产品OpenOffice，甚至开放了其核心操作系统Solaris。Sun也是许多开源组织的重要赞助者。Oracle公司后来全面转向Linux平台，并获得巨大的成功，现在已经成为Linux平台上的最大数据库软件厂商。Intel、Borland、HP、BEA、SAP、Sybase、Adobe等软件公司也不断为开源软件添砖加瓦，在提供资金、设备的基础上，给予技术支持、市场推广等各方面的帮助。这本身也表明了企业对开源模式和开源产品的信任与认同。
迫于安全、反垄断等各方面的压力，Microsoft也开始逐渐有条件地开放其操作系统源代码（仅对政府等特定客户），虽然并非真正意义的开源，但也算是迈出了可喜的一步。Microsoft首席技术官克雷格·蒙特（Craig Mundie）对此表示：“源代码的开放可增进一些客户对我们公司和产品的信任”。Microsoft也积极参与甚至赞助Linux World、Open Source Business Conference（Microsoft为其白金级赞助商）等重量级开源会议。
企业的根本目的在于获取利润，它们参与开源活动的根本动机也是利益驱动的。企业参与开源不会是为了自由、开放的理想，而是为了提升自己的竞争力，在激烈的市场竞争中保持领先地位，最终获取更多利润。通过参与开源活动，无论是传统的企业还是新兴的开源企业都可以获取很多好处，比如提高自己软件产品的质量、降低软件成本、抢占市场份额等等。
因为是开源产品，因此企业一般不太可能通过直接贩卖其开源软件产品获取利润，RedHat公司的主要收益来源是为企业提供服务。参与开源活动的企业除了通过出售技术支持服务（Redhat、JBoss）、相关收费软件（Oracle）乃至硬件（IBM、HP）等开源软件周边产品，它们还有很多其他获利方法，好处甚至大于传统商业模式的经营。当然并非所有参与开源的企业都能成功。
著名的开源项目如Android、Linux、TensorFlow 等，背后多为Google、Microsoft等大型科技公司。近年来国内华为、腾讯、阿里等企业均积极主动发起开源项目，主要集中在云计算、大数据、存储、运维等领域，在国际上的影响力逐渐提升。

### 商业公司参与开源的动机

无论是传统企业还是新兴的开源软件企业参与开源活动的动机主要有：降低成本，建立开放标准，快速抢占市场，提高信任度，战略考虑，实现兼容性，对未来投资等等。
1）降低成本。对开源公司来说，通过开源的理念建立商业项目，不但可以聚合已有的开源构件来完善和丰富项目本身的功能和底层结构，而且开放的源代码可以增加项目的透明度，降低开发风险。对于传统企业，在产品中充分利用开源组件，由于不必开发或者购买这些组件，可以大大降低成本。
2）建立开放标准。软件行业由于产权保护、利益冲突等原因形成了太多的标准，混乱的标准大大增加了企业的支持和维护的成本。利用开源软件可以建立开放、统一的标准，这个标准不为任何组织所独享。以操作系统为例，在Linux获得广泛应用之前，服务器端常见的操作系统是UNIX。UNIX变种极多，它们适用于不同的硬件平台，需要搭配不同的应用软件，操作系统还需要不断升级，由于标准不统一带来很多混乱，无论用户还是软硬件制造商对此都无法容忍。因为在一个特定的系统平台上投资研发实在是风险太大了，尤其对与小公司来说，为产品选择一个错误的平台可能导致公司倒闭。反过来，如果公司所依赖的平台一旦成为标准，在其上进行研发风险则会大大减少，可以使它们专注于解决客户的需求，比其他公司领先一步完成产品，此外它们也可以通过参与标准制定使标准尽量符合它们的现有技术。
3）快速抢占市场。商业产品推出时都一般都需要付出大量的市场推广费，为的是让商品被市场了解。开源项目在整个开发过程中都处在一个透明和开放的环境中，在这个从无到有的过程中，参与者们实际上又是项目的推广者，他们会不遗余力地推广自己的产品，何况开源产品还是免费的。开源软件采用了免费分发、网络发布等形式，比传统的模式减少了营销与销售成本，更易于广泛传播。这些都使得开源产品能够快速地占领市场。
4）提高信任度。开源开发模式在提升软件质量上的作用有目共睹。实践证明，Linux、Apache HTTP Server的安全性、稳定性远远优于Windows和IIS，已经给人们留下了深刻的印象。新兴开源公司的产品因为其开源模式，吸引了许多开发者和用户，由于这些人的关注，使它的安全性、稳定性大大提升，给用户以可信任的形象。无论对开源公司还是传统企业，都可以借助开源提高自己的可信任度。经Microsoft调查，虽然要求开放源码的呼声极高，但真正查看甚至修改源代码的人非常少，只有5%的组织确实查看了源代码，动手修改更是不到1%。对大多数人来说，阅读甚至修改源代码实在令人望而生畏，从成本和收益角度来说也不划算。然而，大部分人还是认为能够访问源码对他们来说非常重要。原因很简单，就是透明度可以提高信任度。
5）战略考虑。部分传统企业参与开源活动是为了实施公司的一些战略计划。例如当年SUN通过发布OpenOffice开源项目来打击Microsoft Office软件在办公市场的地位，顺便提升自己在开源社区的影响力。IBM、HP等服务器供应商巨头，通过捆绑免费的Linux操作系统提高硬件服务器的销售量，这些都是出于公司战略考虑。
6）对未来投资。很难预判未来开源软件将在软件产业中将占据怎样的地位。即使开源软件的未来不像它的支持者描述的那样美好，只要它在未来的软件产业中占据相当的地位，就没有企业可以忽略它的存在。许多企业担心，如果开源大有前途的话，对开源毫无准备的他们可能会丧失发展良机、损失客户，甚至因为缺乏竞争力而损失惨重。许多企业因此而参与开源活动。
7）开源许可证策略。开源软件需要明确许可证，这个许可证规定了参与者的行为。目前开源许可证非常之多，不同的许可证对软件代码使用、发行以及权益归属均有实质性差异。所以如果要发布开源项目，选择合适的开源许可证是一件十分重要的事情。企业在参与开源活动时，有它们的参与动机，为此它们会仔细研究相应的开源许可证是否能满足自己的要求。有时企业在发布新项目时为了自己的利益会制定一套新的许可证协议。
8）市场策略。企业参与开源活动的重要前提是开源项目不能影响其主要利润来源，一般来说就是开源项目与其主要利润点功能不重叠。SAP、Oracle、IBM等公司都表示支持开源软件，他们可以借Linux等开源产品来降低成本、扩充市场份额，但当开源软件涉及到ERP、CRM、中间件、协同商务等它们主要利润来源的时候，它们就会态度大变。
9）盈利策略。硬件厂商IBM、HP、SUN等公司参与开源软件的策略是：它们是大型服务器主机提供商，它们的主机使用开源软件，可以降低整体成本。传统软件厂商如Oracle，它们的策略通常也是利用开源降低成本。新兴的开源企业盈利策略一般有以下几种，而一家公司可能会同时采用其中的几种策略：
- 分类产品线策略。利用开源软件为直接产生收入的专有软件创造或维持一种市场地位。例如，开放源代码的客户端软件带动服务器软件的销售，或者借用开源版本带动商业许可版本的产品销售。这种模式应用得比较普遍。Redhat自Redhat Linux 9.0 后将原桌面操作系统转为 Fedora 项目，借 Fedora Core Linux 在开源社区的声望促进 Redhat Enterprise Linux服务器产品线的销售。
- 授权收费策略。MySQL提供两种许可证：一种是任何软件开发者或组织都可以免费下载MySQL开源数据库，但使用者利用MySQL开发出来的产品也应该与MySQL开源数据库共享。另一种就是付费模式：开发者交了钱后，他们利用MySQL开发的新产品就与MySQL没什么关系了。第一种模式能够使MySQL开源数据库的综合性能逐渐得到提高，第二种模式能为MySQL带来一定的利润。
- 技术服务策略。开源公司利用开源软件先天的传播优势，以极低的成本建立和传播品牌，通过向用户提供产品相关的服务来获得回报。JBoss就是这种模式的典型代表，它的应用服务器完全免费，通过提供技术文档、培训、二次开发等技术服务来盈利。
- 附加产品策略。出售开放源代码的附加产品。比如出售开源技术方面的文档、书籍，O‘Reilly公司就是销售开源软件附加产品成功的典型案例，它出版过很多优秀的开源产品的参考资料。
10）基础研究策略。尽管开源软件是开放、免费的，但是真正要在实际中使用它们还需要深入研究其具体实现，并了解其优缺点，这需要实际参加它的开发过程。“纸上得来终觉浅，绝知此事要躬行。”在产业残酷竞争中，参与开源活动的一个策略就是将其作为一项基础技术研究来进行。
11）知识产权策略。知识产权的问题不仅是一个严肃的法律问题，还会直接影响到一个公司的商业信誉和客户信心。关于开源的知识产权管理是许多开源企业目前迫切需要解决的一个问题，知识产权像一把“双刃刀”，过于严密则影响创新和效率，而过于宽松又会降低竞争力，平衡地规制知识产权的保护和使用是企业参与开源活动的一个重要策略。

### 商业公司参与开源的方式

企业参与开源活动的方法是多种多样的，如参加开源者组织的会议，或重量级产品作为开源项目。一般来说，企业参与活动的方式与动机紧密相联，有什么样的动机，就会有相应的参与方法。
1）提交开源项目。这是新兴开源企业最常见的做法，也是传统企业参与开源活动比较深入的做法。开源项目没有规定大家不能借此盈利，谁都可以从开源的项目获取回报。由于开源软件在易用性、针对性上有其本身固有的弱点，在开源软件基础上进行二次开发，提供自己的非开源产品无可厚非。同时开源软件还需要技术支持，有培训、安装、配置等方面的服务要求，这些服务都不是免费的。利用开源软件盈利的方法和利用传统的封闭软件盈利的方法基本上是一样的，建立一个产品并发布它，为客户提供服务，将其建成一个优秀的品牌。
一些新兴的小型企业，将自己的项目设置为开源项目，充分利用开源的优势，通过提供各种技术支持获取盈利，比较成功的有JBoss、Mysql等软件公司。事实上相当一部分开源项目都是这样形成的。
一些传统企业出于各种原因也渐渐加入这个行列，比较著名的有IBM提交的Eclipse开源项目、SUN提交的OpenOffice开源项目。但是它们的动机就不仅仅是为了直接从中盈利，往往还有其他意图。例如IBM除了可以对Eclipse进行二次开发实现WSAD产品外，还可以借此推广Websphere，并为Rational系列产品提供一个IDE平台。
2）参与开源会议。开源参与者是分散在世界各地的，面对面接触的机会往往就是开源组织或者某个企业召集的大会。这些大会经常讨论开源软件的各个方面的问题，如技术、经济、组织、管理等等，对开源运动十分重要。企业可以直接自己举办这样的开源会议，也可以赞助或参加其他组织举办的开源大会。这是一种浅层次的参与。
3）资助开源活动。为开源活动提供各方面的资助是企业参与开源活动的一种相当普遍的方法，采用这种方法的一般都是传统企业。许多开源项目都得到过这方面的资助，比如Linux、Apache等接受过IBM、Intel等公司的大量资助。企业对开源组织的资助有多种形式，给开源组织提供资金是直接的经济资助；为开源社区提供软件和硬件是间接的经济资助；为开源社区开放专利授权、捐献源代码、提供技术人员参与开源项目等，是对开源的技术赞助。
4）采用开源产品。在自己的产品中采用开源项目，许多企业利用这种参与方法取得了相当好的效果。这是一种双赢的选择，企业通过采用开源项目实现了建立开放标准的愿望、得到了降低成本的实惠，而开源产品则得以进入实际应用，被迅速推广。
5）兼容开源产品。一些传统企业注意到开源产品市场越来越大，就将其部分产品移植到开源产品平台上，或者实现与开源产品的互操作。随着开源市场的不断扩大，越来越多的企业会注意到这个市场，就是最保守的企业也得考虑是否要让自己的产品与开源产品相互协同。
6）提供技术咨询与服务。一些企业通过为开源产品提供技术咨询和服务来盈利，或将开源软件封装成商业发行版方便开源软件的安装与使用。RedHat是就这样起步的。再如Ubuntu的收费服务。为Ubuntu提供技术支持服务的公司是Canonical，其收费标准是全球统一的。对于桌面版的Ubuntu，如果是每周（五天）早上9点到下午5点的技术服务，年度收费标准是144.74欧元；如果是7*24小时服务，年度收费标准是521.05欧元；对于服务器版的Ubuntu，如果每周（五天）早上9点到下午5点的技术服务，年度收费标准是434.21欧元；如果是7*24小时服务，年度收费标准是1870.72欧元。尽管这些服务收费并不比购买商业软件便宜，但实践证明这种需求是客观存在的。

据2015年的调查数据，有78％的受访者表示，他们的公司在OSS上开展部分或全部业务；有66％的受访者表示，他们采用开放源码的方式进行软件开发。有64％的公司目前参与开源项目；在未来2-3年88％的公司有望增加对开源项目的贡献。
尽管目前开源运动如火如荼，我们并不认为开源模式能够完全取代传统的闭源模式（不开源的商业模式）。过去人们谈到开源，就把它放在商业软件的对立面上。随着开源运动的普及和深入，随着开源软件的发展，这种认识已经不合时宜了。开源软件并不能消灭闭源软件，开源模式只是软件的另一种生存形式，由此可能会形成一个与传统的软件产业不一样的开源软件产业。著名开源软件公司Sendmail创办人格雷格·奥尔森（Greg Olson）曾说：“我们并没有抱着要将世界改变成开源的政治任务”。开源和闭源各自有其擅长的领域，它们相互补充，各得其所，共同促进着软件产业的发展。
在目前的市场上，软件开发、授权许可以及业务策略都渐渐进入一种混合模式，很少有纯开源或者纯闭源的企业，实际上和开源运动有着密切关系的大多都是企业。企业参与开源运动势不可挡。开源社区在技术发展方面起到决定性的作用，企业则通过资金支持以及产品化、市场化，极大地推动了开源技术的发展，同时企业自己也获取了相应的回报，这何尝不是一种和谐的发展模式呢？

### 阅读：Microsoft与开源的恩恩怨怨

1975年，Microsoft公司成立，他们推出的第一个产品Basic编程软件深受人们的喜爱，几乎成为每台电脑上必备的软件。由一群计算机爱好者和发烧友组成的“家用计算机俱乐部”是当时最著名的社区之一，俱乐部有一个不成文的规定，会员可以从俱乐部带走软件的磁盘，只要下次见面时带回两份拷贝给其他会员就行了。这是当时人们习以为常的软件传播方式，Microsoft的Basic程序就这样被安装在了许多俱乐部成员的电脑上，而实际上只有原始的拷贝是从Microsoft那里付了钱买来的。1976年比尔·盖茨（Bill Gates）写了一篇文章《致业余爱好者的公开信》，他在文中对业余者大量使用Microsoft的Altair BASIC产品而不支付费用表示愤怒，指责他们说：“你们大多数人都是在剽窃软件”，他质问道：“谁能够承担得起没有收入的工作？什么样的爱好者可以让3个人长年累月无偿地投入编程工作中，找出所有的错误，编制产品手册并进行发布呢？”盖茨呼吁把这些窃贼踢出电脑爱好者的阵营。私有软件和开源软件从此成了势不两立的仇家。
1998年10月，雷蒙德披露了两份Microsoft内部机密备忘录《万圣节文件》。这些文件对开源软件进行了评判，Linux操作系统被认为是对Microsoft主导的软件业的主要威胁，文件还建议Microsoft应该如何抑制开源软件的发展：“……开源软件至少（如果没有超过的话）具有生命力……开源软件内在的平等关系和自由思想对于我们目前的授权模式来说，具有无法复制的好处……开源软件通过互联网聚集和利用成千上万个人和集体的智商的能力简直令人惊奇。更为重要的是，开源软件在互联网上狂热的传播要远比我们自己的努力传播来得快。……是否有可能在不失去控制的情况下引入开源的核心元素?......是否能够从开源运动中学会如何更好地分享个人创意，处理沟通问题和复杂情况，并设计可以更好地回馈协作者的机制？或者，我们能否通过挖走某些程序员，散布有关开源会带来恐惧、不确定和疑虑的言论，并利用和其他手段强化知识产权，以达到削弱开源进程，进而达到扼杀开源的目的?”显然，这些文件既承认了开源产品在技术上是Microsoft产品的有力竞争对手，又极尽破坏开源之能事。这些文件使Microsoft相当尴尬，因为跟Microsoft以前在这方面的表态自相矛盾。
面对开源软件的汹汹来袭，Microsoft开始了它的反击。2001年它推出了一项“共享源代码”计划，依据这项计划，独立开发者在开发系统级的软件时可以申请查看他们所需的源代码，政府和单位出于信息安全的考虑也可以要求审查源代码，而教育研究机构在进行科研时亦可以申请获得源代码的支持。在外界看来，这也许是Microsoft向开源模式低头的表现，但Microsoft内部依然是抗拒开源模式的。Microsoft高管蒙特在纽约大学商学院的一次演讲上说：“共享源代码计划仅仅是吸收了开源模式的优点，例如提供有效反馈以及Bug修复、可定制等，而实际上，（开源模式）要求开发者把他们辛勤工作产出的最具价值的结果无偿提供给人们，转而希望他们能通过提供一些诸如技术支持之类的其他东西来养活他们自己，这是对独立软件持续发展的最大扼杀。”虽然受到不少批评，但“共享源代码”计划时下依旧在运作着，其间还推出过获得OSI认证的MS-PL授权许可。
两年后，为了推广精简版的Windows XP，盖茨多次访华，同时与中国政府签署了“政府源代码备案计划协议”，盖茨还亲自向江泽民汇报。从Microsoft的资料来看，政府源代码备案是一项全球性的计划，旨在让政府和备案单位通过可控的方式查看Microsoft的源代码和相关技术信息，使政府部门得以对Windows的安全性做一个全面的评估，强化双方的合作。只不过，Microsoft方面并没有对“可控的”一词做出进一步解释。
2005年以后，Microsoft对开源的态度有所转变。一方面，Microsoft向开源阵营摇动橄榄枝，捐赠代码，和越来越多的开源厂商签署互操作协议，例如， RedHat就与Microsoft签署了合作协议，加强虚拟化平台互操作；另一方面，它仍然对那些基于开源技术的专利实行收费，还动辄就跟开源软件对簿公堂，声称开源软件侵犯了Microsoft的专利。Microsoft在2007年曾公开表示，开源软件侵犯了它的230多项专利。其中，Linux内核侵犯了42项，Linux用户界面和其它设计元素侵犯了65项，OpenOffice.org侵犯了45项，其它开源软件侵犯了83项。Microsoft此举在当时引发业内恐慌，同时也遭到了开源社区的一致谴责。但Microsoft后来又重新向开源示好，2010年4月Microsoft美国市场技术高管斯图亚特·麦基（Stuart McKee）表示，Microsoft已经改变了此前对开源社区的敌视态度。他承认，Microsoft此前与开源软件界存在对立关系，但如今已经调整了这种关系。
Microsoft在2014年的云策略发布会上大胆喊出了“Microsoft 爱 Linux”的口号，这在史蒂夫·鲍尔默（Steve Balmer）掌权Microsoft的时代是难以想象的，在他口中“开源就是一个毒瘤，谁碰上了都要倒霉。”那么这对昔日的仇家如今怎么走向合作了呢？新任CEO萨提亚·纳德拉（Satya Nadella）的上台及其提出的新的发展战略是一个关键的转折。
在盖茨和鲍尔默时代，Windows被视为Microsoft的业务核心，他们要做的就是如何让客户更多地购买他们的软件。在他们的思维中互联网是次要的，IE浏览器糟糕的体验就一直被人吐槽。一晃二十多年过去了，Microsoft赢得了桌面操作系统的胜利，Windows操作系统依然运行在数以万计的PC设备上。但今天是一个移动互联网的时代，PC市场正在不断地萎缩，与此同时，Apple和Google这样的竞争对手逐渐崛起，他们在互联网、移动互联网、云服务甚至物联网构筑起成熟的生态，而Linux则在服务器及超级计算等高精尖领域把Microsoft压得抬不起头。醒悟过来的Microsoft亡羊补牢，可惜为时已晚，移动互联网市场大势已去。
也许是在互联网和移动计算时代输得太惨，纳德拉接替鲍尔默执掌Microsoft后，Microsoft的战略从模仿Apple的“设备和服务”变成了“移动优先云为先”。纳德拉的第一个举动就是将Microsoft云计算服务Azure之前的“Windows”字样果断去掉，在此之前Azure平台只能运行在Microsoft自家的Windows Server操作系统上。在他看来，无处不在的“Windows”印记，是PC时代的落后思维，在云计算和移动互联网时代，Microsoft的云服务不应该再束缚于Windows的高墙之内。随后，Microsoft陆续与相关公司开展合作，将Debian Linux、Red Hat Linux和Ubuntu Linux一批最先进的Linux发行版本带到Azure平台上。目前，在Azure云平台上已有50%以上的虚拟机运行着Linux，用他们的话来说，“识时务者为俊杰”，这些举动让有特殊需求和喜好的客户有更大的余地来选择产品，对提高Microsoft自身的竞争力无疑是有帮助的。
实际上，早在2009年Microsoft就曾向Linux内核贡献了超过两万行代码，这些代码大大增强了在Windows Server 2008 Hyper-V上运行Linux操作系统的虚拟机的性能，Microsoft甚至一度在Linux内核贡献榜上排名第一。
除此之外，Microsoft还在开源这条路上走得更远。Microsoft为开源的开发者提供开源、免费的代码编辑器Visual Studio Code，支持主流的30几种编程语言，任何人可下载其源代码并依个人喜好进行定制，可在Windows、Linux与Mac环境上运行。Microsoft还将Windows下广受开发者喜爱的应用框架.NET Framework的核心部分开源，称为.NET Core，支持Windows、Linux与iOS不同的操作系统，除了开源还能跨平台，已经成为GitHub上最受欢迎的开源项目；在收购了Xamarin后，Microsoft宣布将这一移动跨平台开发技术免费开放给开发者使用，以实现“一次开发，多平台部署”，单一开发者就能够开发iOS、Android与Windows Phone的原生App，企业不再需要为一个App养活3个开发团队了。另外，Microsoft具有竞争力的数据库产品也推出了SQL Server 2016 For Linux版本，为服务器间建立一致的数据平台带来便利；Ubuntu Bash on Windows项目则将Linux上最重要的命令行组件带到了最新的Windows 10上，这意味着大量Linux上的开源开发工具可以直接运行在Windows上面了。
在跨平台方面也可以看得出Microsoft所投入的心血。Office原本是作为Windows的杀手锏之一，但是人们看到Microsoft推出了Android和iOS版本的免费Office。甚至，在iPad Pro的发布会上，Microsoft的高管还应邀为对手站台，现场演示iPad版的Office应用。Cortana智能语音助手，Outlook邮箱，必应输入法也纷纷登陆到了对手的平台上，使更多用户能在不同的平台上享受Microsoft一致的服务体验。
2014年9月，Microsoft预览体验项目（Insider Program）随着Windows 10的第一个技术预览版本一起面世，致力于为喜欢尝鲜的用户推送包含最新功能、处于测试阶段的Microsoft产品，并通过一个“反馈中心”的程序让用户提交他们的使用体验及错误报告。
纳德拉说：“营收属于落后指标，而使用率才是领先指标。”在这个思维的引导下，Microsoft大力拥抱开源，培养多元化的生态系统，从Microsoft云Azure到Windows，从开发工具到数据库平台，这些举动表明Microsoft不再死守Windows不放而选择了开放。当然，这也给它带来了新的机遇。开源意味着它能够收到来自社区的反馈和改进，而跨平台则使其能够向更多的用户兜售其服务，以Office为例，其收费模式已经从传统的“一次购买，终生使用”，转向包月、包年购买使用权了，从长期看这无疑有益于提高其收益。
开源软件界对于Microsoft的开源行动一直保持清醒的头脑，由于Linux平台的用户数量庞大，Microsoft意识到如果他们不能提供Linux虚拟机，那些客户就会转而选择其他厂商。正如美国的一句俗语说的“如果无法战胜你的敌人，那么就拥抱他吧”。说到底，Microsoft还是那家商业利益至上的公司。Microsoft的行动引发了一些担忧：“.NET Framework确实是一个优秀的框架，在跨平台的开源模式下将帮助其更加完善，进而吸引越来越多的开发者使用它。但是，开源社区的一个很明显的问题是，很多项目都活不长，开发者在丢出最初几个版本后就销声匿迹的项目比比皆是。.NET的广泛采用可能意味着与之竞争的开源项目会走向消亡，最终迫使我们都跟着Microsoft的指挥棒转。”从这个角度看，Microsoft的开源行动对于开源社区来讲也许不是什么好消息。
随着Microsoft开始认可更多的开源软件，Microsoft在开源方面的立场已经发生了变化。2016年，Microsoft前首席执行官鲍尔默撤回了有关Linux是恶性肿瘤的声明。在2017年，Microsoft成为Linux基金会的白金级支持者。到2018年，在收购GitHub之前不久，Microsoft一般以上的有薪员工都为开源项目做出过贡献。10月26日，Microsoft通过75亿美元的股票交易收购了全球最大的开源软件托管平台GitHub，Microsoft开发者服务副总裁奈特·弗里德曼(Nat Friedman)成为GitHub的新一任CEO。

### 开源软件的未来发展

据统计，当前的开源程序及项目多达30万种，可以通过5000个网站进行下载。仅Google的开源项目就超过了2000个。截至2018年，全球最大的代码托管平台GitHub已有3000万开发人员，其中2018年的新用户数量超过了前六年用户数之和；囊括200万家企业或组织，2018年的组织数目比上一年增加了40%；拥有9600万个代码库，超过三分之一的代码库是在2018年创建的。开源软件已成为每个软件公司的一部分。
中国贡献者在开源社区中持续活跃。截至2018年，全球最大的代码托管平台GitHub上超过80%的用户来自美国以外的地区，其中中国的贡献者数目仅次于美国，排名第二。2018年GitHub上的贡献者数量是2017年的1.6倍，其中中国的新注册用户数目仅次于美国，排名第二。我国科技类公司在跟随国际顶级开源项目的同时，积极推广自己的开源项目，阿里、腾讯、华为、滴滴等科技公司纷纷成立开源管理办公室，负责公司对外开源的统筹规划，进行开源之前的合规检查及后续运营推广。截至2019年，阿里已对外开源150个项目，腾讯已对外开源69个项目。
站在2019 年，可以说没有开源软件就没有软件业的今天。几乎所有与我们交互的计算机系统，至少都部分地使用了开源软件。从宏观上看，开源软件不仅催生了一大批创业公司，还成就了全球史上最大的一宗软件收购——IBM收购Red Hat。
随着下一代互联网、移动互联网的发展，开源软件的理念将与软件服务化的理念深度融合，开源运动将持续推动软件产业的创新，开源软件还将迎来一轮轮的发展高潮。从开源软件的发展历史，我们能够初窥开源软件未来发展的端倪：
1）云计算将与开源操作系统和应用软件紧密结合，以“软件即服务（Software as a Service, SaaS）”的方式提供用户使用。一方面，云计算和SaaS初始成本很低，可以帮助用户降低成本，使开源软件的企业实现盈利。另一方面，多用户租赁云端的开源软件，通过开放的应用编程接口进行控制，可以显著降低安装和维护的成本，对于开源技术的大规模应用能够起到拉动作用。例如，国内大量的电子政务应用建立在共有云等资源之上，为社会提供服务。有人预言，未来的世界将会是“一切皆服务（Xaas）”，开源也将无所不在。开源软件界更加希望云服务商在使用开源软件的同时能够对开源多做贡献。
2）开源软件将进一步深入关键应用领域。虽然目前银行、机场、海关等关键应用领域还是商业软件的领地，但是支持这些关键应用的核心技术越来越多地来自开源软件，已经足以证明开源软件的安全性、可靠性和稳定性。将有更多的渠道商和增值服务商、系统集成商和其它服务提供商，把开源技术整合到它们的产品或服务中，导致越来越多的关键应用领域应用基于开源软件的服务。
3）开源软件将继续在人工智能领域大展宏图。人工智能的应用范围将进一步扩大。一些新领域，如数据准备、集成、算法选择、训练方法选择和模型创建，都将借助机器学习来进行。新的开源智能解决方案将改变人们与系统交互的方式，并将带来工作性质的转变。
4）开源将不断成为新技术的发源地。开源推动创新是当今信息技术产业的常态。Redis，MongoDB，Spark，Docker ……这些带来创新理念的基础软件，无一是按照传统商业软件的模式运作的。今天开源成为大多数技术创新背后的驱动力。例如，区块链从比特币走了很长一段路。这项技术已经在金融、安全投票、学术证书认证等领域得到广泛应用。区块链分发数字信息，信息驻留在数以百万计的节点上，不受任何集中站点的控制，也没有单一的故障点，使得它非常强大、透明和廉洁。它还解决了中间人操纵数据的威胁。这种内在的优势决定了区块链在不久的将来将成为一种主流技术。

### 开源运动将大大推动软件产业的创新，关键的开源项目将引领时代。

5）开源软件将与商业软件长期共存。混源（既有开源软件也有闭源软件）业务在不断增长，很可能成为主流的商业模式——开源软件厂商销售衍生软件，同时闭源厂商提供更多的开源软件。所有的软件表面上看起来都是开源的，他们使用一种复合许可证，你可以像使用开源软件一样使用它，但是可能要为底层开发者支付一些费用。
6）软件的安全性和可靠性将始终是人们的关注的焦点。这也是开源软件的优势所在，持续提高软件的安全性仍然需要开源软件界的不懈努力。
