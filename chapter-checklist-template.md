## Round 1：核心章节开发

### 内部章节包

- [ ] **本章定位**：明确本章在全书中的作用、与前后章的逻辑关系、主要解决什么问题
- [ ] **中英文标准标题**：确定本章的中英文标准标题和英文短标识
- [ ] **学习目标**：3-6 条可观察、可评价的学习目标
- [ ] **基线层内容**：明确标注哪些内容属于基线层
- [ ] **扩展层内容**：明确标注哪些内容属于扩展层，且不打乱正文节奏
- [ ] **正文主叙述结构**：3-6 个一级小节，叙述主线清晰
- [ ] **核心术语表**：中文术语、英文原词、推荐译法、一句话解释、本章语境
- [ ] **案例接口**：明确本章使用什么类型的案例、案例说明什么问题、放在哪里
- [ ] **附录接口**：标记哪些内容应下沉到附录，写清需要什么附录支持
- [ ] **实验接口**：说明本章如何进入实验体系、对应哪个实验阶段、需要什么交付证据
- [ ] **本章可评价证据**：明确教师通过什么证据判断学生是否达到本章目标
- [ ] **参考文献**：核心参考文献、推荐扩展阅读、案例来源

### 技术书稿

- [ ] **书稿正文初稿**：按技术书标准写作，清晰连贯，学生可读
- [ ] **开场切入**：用真实问题、仓库场景、历史事件或工程困境引入
- [ ] **案例贯穿**：案例在正文中承担解释功能，而非仅在章末附属
- [ ] **关键提醒**：适当位置加入 Note / Tip / Warning
- [ ] **本章小结**：收束核心认识，自然连接下一章
- [ ] **延伸阅读**：章末或版式化处理
- [ ] **术语一致性**：与全书术语表一致，无随意漂移

## Round 2：教学支持层

- [ ] **Student Study Guide 映射**：推荐阅读路径、重难点、误区、与项目主线连接、自查问题
- [ ] **Instructor Guide 映射**：教学重点、课堂组织建议、案例选择、实验映射、评价证据
- [ ] **Course Syllabus 映射**：标注本章对应课程第几周/模块、与里程碑和考核节点的关系

## Round 3：支撑材料

- [ ] **附录内容**：命令速查、模板、配置示例等已产出或已标记为待补
- [ ] **案例库条目**：本章调用的真实仓库、事件、治理案例已整理或已标记
- [ ] **实验材料**：本章对应的实验说明、任务要求、评分标准已产出或已标记

## 整体检查

- [ ] **视角一致性**：本章内容是否从蓝图标注的视角出发，未与其他章节视角重复
- [ ] **与 00-preface 对齐**：本章内容是否符合 00-preface 的课程定位和写作原则
- [ ] **双语可映射**：章节标题、术语、栏目名称可稳定中英文对应
